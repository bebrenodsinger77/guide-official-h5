# guide-official-h5

## 项目简介

本仓库主要用于归档和发布多个独立的 HTML 页面。这些页面可能包含工具类、展示类、交互类或信息类内容，以静态 HTML 形式存储，不依赖特定后端服务。

仓库本身不针对任何具体网站或域名，仅作为通用 HTML 页面集合的托管与发布场所。

## 目录说明

```
guide-official-h5/
├── index.html          # 入口页面，可查看所有归档页面的链接列表
├── pages/              # 存放各类独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 公共资源文件（CSS、JavaScript、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `index.html`：建议作为仓库的默认展示页，提供页面导航或说明。
- `pages/`：所有归档的独立 HTML 文件存放于此，可按功能或主题命名。
- `assets/`：存放页面共用的样式、脚本、图片等资源，便于统一管理。

## 页面归档说明

- 每个 HTML 文件应保持独立，不依赖仓库外的资源（除非必要）。
- 文件名应清晰反映页面内容或用途，避免使用无意义编号。
- 建议在页面内添加简短的标题或说明，方便直接浏览时理解用途。
- 若页面需要外部资源（如字体、库文件），请尽量使用 CDN 或放入 `assets/` 目录。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并在 `index.html` 中添加对应链接。
- 修改页面：直接编辑对应文件，注意保持与其他页面的兼容性。
- 删除页面：移除文件并更新 `index.html` 中的链接。
- 资源更新：修改 `assets/` 下文件时，请确认所有引用该资源的页面均能正常工作。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任意 HTML 文件。
3. 也可通过 GitHub Pages 或其他静态托管服务发布，以便在线访问。

## 注意事项

- 本仓库仅为静态页面集合，不包含后端逻辑。
- 请勿在页面中嵌入任何恶意代码或违反平台政策的内容。
- 所有页面内容由提交者自行负责。
